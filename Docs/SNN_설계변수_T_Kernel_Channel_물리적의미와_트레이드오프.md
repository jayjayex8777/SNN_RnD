# SNN 설계 변수 T, Kernel, Channel의 물리적 의미와 트레이드오프

## 1. T (Timestep) 가변

**물리적 의미**: 입력 신호를 스파이크로 변환하여 **몇 번 관찰할 것인가**

Rate coding에서 원래 IMU 신호의 크기는 스파이크 발화 확률로 변환됩니다. 예를 들어 가속도 값이 크면 발화 확률이 높아지고, 각 timestep마다 확률적으로 0 또는 1이 결정됩니다.

- **T=3**: 스파이크 패턴을 3번만 관찰 → 발화율 추정이 거침 (동전을 3번만 던지는 것)
- **T=20**: 20번 관찰 → 발화율 추정이 정밀 (동전을 20번 던지는 것)

또한 LIF 뉴런의 막전위가 T step 동안 누적/감쇠하므로, T가 길수록 시간적 맥락(temporal context)을 더 많이 반영합니다.

**장점 (T 증가)**:
- 발화율 추정 정밀도 향상 → 정확도 상승 (대수의 법칙)
- LIF 뉴런이 더 많은 시간적 패턴을 통합 가능
- 노이즈에 강건 (확률적 스파이크의 분산이 줄어듦)

**단점 (T 증가)**:
- **Latency가 T에 비례**: 매 timestep마다 Conv + LIF 전체를 반복 실행
- **에너지가 T에 비례**: MAC/AC 연산 횟수 = T × (1 step 연산량)
- 입력 텐서 크기 증가: (B, 12, N, T) → 메모리 부담

**핵심 트레이드오프**: T는 **정확도 vs 속도/에너지**의 직접적인 조절 변수

---

## 2. Kernel Size 가변

**물리적 의미**: Conv1d가 입력 시계열의 **얼마나 넓은 구간을 한번에 볼 것인가**

Conv1d는 각 timestep마다 1D 시계열(길이 N)에 대해 슬라이딩 윈도우로 동작합니다. kernel_size=k이면, 연속된 k개 데이터 포인트를 한번에 보고 하나의 feature 값을 출력합니다.

- **k=3**: 짧은 구간 → 급격한 변화, 미세한 진동 같은 **국소적 패턴** 포착
- **k=11**: 넓은 구간 → swipe의 전체적인 궤적 같은 **광역적 패턴** 포착

IMU 제스처로 비유하면:
- k=3: "손가락이 지금 이 순간 어느 방향으로 움직이나?" (미분적 특성)
- k=11: "손가락이 전체적으로 어떤 호를 그리나?" (적분적 특성)

**장점 (k 증가)**:
- 수용 영역(receptive field) 확대 → 긴 범위의 시계열 패턴 인식
- 두 Conv1d 층을 거치면 수용 영역이 더 커짐 (k=11 × 2층 → 매우 넓은 범위)

**단점 (k 증가)**:
- 파라미터 수 증가: Conv1d 파라미터 = `k × in_ch × out_ch` → k에 비례
- per-timestep MAC 연산 증가
- 국소적 미세 패턴을 평활화(smoothing)하여 놓칠 수 있음
- 소규모 데이터셋에서 과적합 위험

**장점 (k 감소)**:
- 더 적은 파라미터 → 작은 모델, 빠른 추론
- 미세한 신호 변화에 민감

**단점 (k 감소)**:
- 좁은 수용 영역 → 긴 범위 패턴 인식 어려움

**핵심 트레이드오프**: Kernel은 **수용 영역(패턴의 스케일) vs 모델 크기/연산량**의 조절 변수

## 3. Channel 가변

**물리적 의미**: 각 Conv1d 층에서 **몇 가지 종류의 특징을 추출할 것인가**

채널 수는 필터(feature map)의 개수입니다. 각 필터는 입력에서 서로 다른 패턴을 학습합니다.

- **c1=16**: 1층에서 16가지 저수준 특징 추출 (예: 특정 주파수 성분, 상승/하강 에지 등)
- **c1=48**: 48가지 저수준 특징 → 더 다양하고 세밀한 특징 표현
- **c2**: 2층에서 c1의 조합으로 고수준 특징 생성 (예: "빠르게 올라갔다 천천히 내려오는 패턴")

**장점 (채널 증가)**:
- 표현력(representational capacity) 증가 → 더 복잡한 패턴 구별 가능
- 제스처 간 미묘한 차이를 잡아낼 수 있음
- 정확도 향상 가능성

**단점 (채널 증가)**:
- 파라미터 수가 채널의 **제곱**에 비례: Conv1d(c1, c2, k) 파라미터 = `k × c1 × c2`
  - (16,32) → k×16×32 = 512k
  - (48,96) → k×48×96 = 4,608k → **9배 차이**
- 모델 크기, MAC 연산, 에너지 모두 증가
- SNN 특유: 채널이 많을수록 **스파이크 트레인 수도 증가** → AC 연산도 채널에 비례
- 소규모 데이터셋에서 과적합 위험 (7,596개 데이터에 큰 모델)

**장점 (채널 감소)**:
- 작은 모델 → 모바일 배포에 유리
- 추론 속도/에너지 절감
- 과적합 방지

**단점 (채널 감소)**:
- 표현력 부족 → 복잡한 패턴 구별 어려움

**핵심 트레이드오프**: Channel은 **표현력(정확도) vs 모델 크기/에너지**의 조절 변수

## 세 변수의 에너지 영향 비교

SNN의 총 에너지는 다음과 같이 분해됩니다:

```
총 에너지 = T × (MAC_per_step + FR × AC_per_step)
```

| 변수 | MAC 영향 | AC 영향 | Latency 영향 | 모델 크기 영향 |
|------|---------|---------|-------------|-------------|
| **T 증가** | T에 비례 (선형) | T에 비례 (선형) | T에 비례 (선형) | 없음 (가중치 동일) |
| **Kernel 증가** | k에 비례 (선형) | k에 비례 (선형) | step당 연산 증가 | k에 비례 (선형) |
| **Channel 증가** | c²에 비례 (제곱) | c²에 비례 (제곱) | step당 연산 증가 | c²에 비례 (제곱) |

**Channel이 가장 민감한 변수**입니다. 채널을 2배로 키우면 파라미터/연산이 ~4배 증가하지만, T나 Kernel을 2배로 키우면 ~2배만 증가합니다. 이것이 채널 가변형이 CNN과의 공정 비교에 중요한 이유이기도 합니다 — 같은 채널 구조여야 모델 용량(capacity)이 동일한 조건에서 비교 가능합니다.
