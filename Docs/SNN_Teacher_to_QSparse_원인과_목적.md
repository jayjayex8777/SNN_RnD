# SNN Teacher → QSparse 까지의 원인과 목적

핵심 목표는 **모바일/엣지 디바이스에서 실시간 제스처 인식**입니다. 각 단계는 이전 단계의 한계를 해결하기 위해 도입되었습니다.

---

## Phase 1 → Phase 2 (Teacher → Student KD)

**문제**: Teacher의 Soft LIF는 **하드웨어에서 실행 불가**
- Soft LIF의 스파이크가 연속값 (0~1) → 실제 뉴로모픽 칩/저전력 하드웨어는 이진 스파이크(0/1)만 처리
- sigmoid surrogate는 학습에는 좋지만 추론 시 불필요한 연산 부담

**목적**: 이진 스파이크(Hard LIF)로 전환하되, KD로 정확도 손실 최소화

---

## Phase 2 → Phase 3 (Student → Sparse)

**문제**: Hard LIF Student의 **발화율이 80~95%로 너무 높음**
- SNN의 에너지 절감 원리: 스파이크가 발생한 뉴런만 연산 (AC 연산) → 발화율이 낮을수록 에너지 절감
- 발화율 80~95%면 거의 모든 뉴런이 매번 발화 → CNN 대비 에너지 이점 없음
- SNN을 쓰는 의미가 없어짐

**목적**: 발화율을 5~30%로 낮춰서 **SNN 본연의 에너지 효율** 확보
- 발화율 5%면 → 95%의 뉴런이 쉬고 있음 → AC 연산만으로 처리 → 에너지 대폭 절감

---

## Phase 3 → Phase 4 (Sparse → QSparse)

**문제**: Sparse Student의 Conv1d 가중치가 여전히 **FP32 (32bit)**
- 모바일 디바이스 메모리/저장 공간 제약
- FP32 MAC 연산 에너지: 4.6pJ vs INT8 MAC: 0.2pJ → **23배 차이**
- 모델 크기도 모바일 배포에 부담

**목적**: Conv1d를 INT8로 양자화하여 **모델 크기 축소 + MAC 연산 에너지 절감**
- LIF 뉴런은 FP32 유지 (막전위 정밀도 필요)
- Conv만 INT8 → 크기 ~2x 압축, MAC 에너지 ~23x 절감

---

## 요약: 단계별 최적화 대상

| 단계 | 해결한 문제 | 최적화 대상 |
|------|-----------|------------|
| Teacher → Student | 연속 스파이크 → 이진 스파이크 | **하드웨어 호환성** |
| Student → Sparse | 높은 발화율 (80~95%) | **AC 연산 에너지** (스파이크 수 줄이기) |
| Sparse → QSparse | FP32 가중치 | **MAC 연산 에너지 + 모델 크기** (비트 줄이기) |

결국 **정확도를 최대한 유지하면서, 모바일에서 돌릴 수 있는 가장 작고 에너지 효율적인 SNN**을 만드는 것이 전체 목적입니다.
